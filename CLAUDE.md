## Core Features (MVP)

1. **Upload** — Drop a video file (root screen, no landing page)
2. **Analyzing** — Real progress: extract frames → Stage 1 (vision) → Stage 2 (text generation)
3. **5-Step Wizard**
   - Step 1: Title selection (4 AI options)
   - Step 2: Thumbnail (4 AI-picked real video frames, enhanced via Canvas, + custom upload)
   - Step 3: Description (single AI-generated text block)
   - Step 4: Tags (multi-select)
   - Step 5: Overall score + export
4. **Export** — Copy to clipboard for YouTube Studio

## Key Architecture Patterns

### State Machine (App Phase)
```
"upload" → user drops a video file (root screen)
   ↓
"analyzing" → real progress across 3 phases:
   1. Extract frames client-side (<video> + <canvas>)
   2. Stage 1: Gemini vision call (frames → description + thumbnail frame picks)
   3. Stage 2: Gemini text call (description → titles, description, tags, score)
   ↓
"results" → 5-step wizard (step 0-4)
```

### Two-Stage AI Pipeline
```
Stage 1 (Vision):  frames[] → { description, thumbnailFrameIndices[] }
Stage 2 (Text):    description → { titles[4], description, tags[], overallScore }
```

### Scoring
A single **static overall score** (0-100) is generated by Stage 2 and shown on the
final step. No live recalculation, no per-item CTR/SEO/trend breakdown.

### Component Organization
```
App (root state machine)
├── Upload          (phase: "upload", root "/")
├── Analyzing        (phase: "analyzing")
└── ResultsWizard    (phase: "results")
    ├── TitleStep
    ├── ThumbnailStep
    ├── DescriptionStep
    ├── TagsStep
    └── RankStep
```

## Important Constraints

- **No video upload to a server** — frames are extracted client-side; only frames/text go to Gemini
- **No video storage** — nothing persisted, metadata only during the session
- **No database** — in-memory React state only, cleared on refresh
- **No accounts, no payment, no rate limiting** — personal use
- **Video limits** — up to 10 minutes / 300MB, validated client-side on Upload
- **Privacy note** — inline note on the Upload screen explaining that extracted frames are sent to Google's Gemini API
- **Error handling** — manual "Try Again" on failure (retries only the failed stage), no mock-data fallback

## Security

This project handles:
- Google Gemini API key (server-side only)
- Video frames sent to Gemini for analysis (no permanent storage)

### Local Development

**NEVER commit `.env.local` to GitHub**

1. Create `.env.local` (ignored by git)
2. Add your `GOOGLE_GEMINI_API_KEY` there
3. Never share this file
4. Delete keys if exposed
